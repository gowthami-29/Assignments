# a. Package Managers:
## Package Manager:
    A package manager is a tool used to install, update, and manage libraries required for a project.
## Need package managers in backend development:
    They help manage project dependencies easily.
    Ensure version control and consistency across environments.
## Problems:
    Manual dependency installation becomes difficult.
    Version conflicts may occur.
    Project setup and maintainance become error-prone.
# b. NPM:
## NPM:
    NPM is the default package manager for Node.js used to manage js packages.
## Importance:
    it provides acess to thousands of resuable packages.Simplifies dependency installation and updates.
## NPM helps in managing dependencies:
    uses pacakge.json to list dependencies.automatically installs required packages.
# c. Backend Project Initialization:
##  command used to initialize a backend (Node.js) project:
    npm init
## npm-init & npm init -y:
* npm -init:
    prompts the user to enter project details and creates package.json
* npm init -y:
    creates package.json with default values without asking questions.
# d. Files and Folders Created After Project Initialization:
## package.json:
    stores  project information , scripts, and dependencies.
## node_modules:
    contain installed packages.
## package-lock.json:
    looks exact versions of installed dependencies.
